# Candidates

## 目的

1. 把當下所有可以做的事情依重要性排序，列出來，讓使用者可以從裡面選擇要做什麼。
2. 因為是把腦袋當作單緒的執行緒，因此，一次只允許一項任務執行，一旦開始了一個任務，畫面就會鎖定在該任務上，直到完成了才會回到可以選擇的狀態。
3. 允許中斷，可透過 API 或者按 ⚡來中斷當前任務，然後改進到中斷任務的任務裡面。
4. 由於中斷任務本身就是個任務，所以，不管有無任務在跑，都可以隨時中斷，等結束時再記錄剛剛為了甚麼而中斷和做了甚麼，放到 `Log` 裡面。
5. 當前任務的狀態會顯示在畫面上，讓使用者知道自己在做什麼。
6. 所以，這是最常使用的頁面。

## 使用方式

1. 選擇任務：
    1. 點擊任務即會跳出對話視窗，紀錄一下為何要開始這個任務，確認後，才會開始執行
    2. 一旦確認，畫面會一直停留在執行此任務的對話視窗，右上角會顯示已經執行多久了，直到使用者點擊 `結束任務`，才會結束此任務，並且紀錄剛剛為何要開始這個任務和做了甚麼，放到 `Log` 裡面。
    3. 若要中斷任務，點擊 ⚡ 按鈕即可，他會結束目前的任務，然後進到中斷任務的對話視窗。這些動作都會記錄到 `Log` 裡面。
2. 🔄 刷新候選：
    1. 點擊 `刷新候選` 按鈕，會重新從資料庫裡面讀取任務，並且根據優先級排序後顯示出來。
    2. 原則上，每當回到這個頁面時，就會自動刷新一下。
3. 快速判斷：
   1. 越上層表示越應該注意
   2. 紅色表示過時了，綠色表示今天差不多夠了
   3. 如果 `Task Pool` 的任務裡面帶有 `⛔` 的話，表示這個任務其實是您不想花那麼多時間的(比如社群軟體)，所以，若使用太多，他會變成灰色外加刪除線，告訴您使用太多了
   4. 每個條目都有寫還是多少時間，方便決定要不要做
5. Deadline（截止日）：
   1. 在 Task Pool、Micro Tasks、Scheduled 各任務中可設定截止日（Deadline）。
   2. 截止日越近，評分越高，任務會排到越前面：7 天內顯示倒數 badge，3 天內以橘色標示，當天以橙色強調，逾期以紅色顯示。
   3. 逾期加分設有上限（+500），因此即便任務已逾期許久，系統也不會強迫您立即處理——這是您的決定。建議定期瀏覽一下，決定這些超期任務要繼續、延期或捨棄。
   4. 若 Scheduled 任務的下次執行時間（Next Run）晚於截止日，候選頁面上方會出現警示區塊，點擊該條目即可直接跳至 Scheduled 編輯頁面修正排程或截止日。
4. iPhone 上捷徑的使用：
   1. 安裝 `NBL Query` 捷徑，就會打開此頁面 (抱歉，由於iPhone 的捷徑無法打開 PWA，因此，他幾乎每次都會開一個新的 Safari 視窗來顯示這個頁面，這是無法避免的了)，這樣就可以快速查看當下有哪些任務可以做，並且選擇要做什麼了。
      1. 建議將他放到 iPhone 的 `控制中心`，這樣由手機右上角往下滑，就可以點選該捷徑，快速打開此頁面了。
      2. 建議將他放到 iPhone 的 `鎖定畫面` 的 `小工具` 裡面，這樣在鎖定畫面就可以看到當下有哪些任務可以做了。
   2. 安裝 `NBL Interrupt` 捷徑，就可以快速中斷當前任務了，這樣就不會錯過了。
      1. 建議將他放到 iPhone 的 `控制中心`，這樣由手機右上角往下滑，就可以點選該捷徑，快速中斷當前任務了。
      2. 建議將他放到 iPhone 的 `鎖定畫面` 的 `小工具` 裡面，這樣在鎖定畫面就可以快速中斷當前任務了。
   3. 安裝 `NBL_Timer` 捷徑，這樣就可以配合 `Apple Clock` 的計時器功能，來利用番茄時鐘管理法來提醒自己該工作了還是該休息了。
