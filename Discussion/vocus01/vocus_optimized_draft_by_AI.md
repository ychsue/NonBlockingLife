# NBL：在信實與愛裡，為人生狀態切換留一個清醒的介面

有時候，我們不是不知道該做什麼。
我們只是在人生回到 idle 的那一刻，太容易被慣性帶走。

明明想做有價值的事，卻滑進社群、娛樂、脾氣，然後一天就過去了。

NonBlockingLife（NBL）想做的，不是把你的一天塞滿。
而是在每次狀態切換前，給你一個有意識的選擇點。

## 生活如同一個狀態機

如果把生活看成狀態機，我們每天都在不同狀態間切換。

- 專案工作是否專注
- 瑣事是否慢慢被清掉
- 排程事項是否在對的時間被處理

當我們回到 idle，看似自由，其實也最脆弱。
沒有提醒機制時，人很容易沿著習慣和欲望走，掉進娛樂、壞習慣、壞脾氣的迴圈。

NBL 的定位就是這裡。
它不是娛樂型 App，而是狀態切換前的行為介面。

## 為何有效：信實與愛

我想把 NBL 的核心濃縮成兩個詞：信實與愛。

信實，是忠實記錄。
開始一個狀態，就記錄開始；離開一個狀態，就記錄結束。
不粉飾，不自欺，先看見真相。

愛，是溫柔調整。
記錄不是為了責備自己，而是為了在偏離時，仍願意回到主線。
即使今天做得不理想，也能繼續前進。

NBL 不是績效工具。
它是幫助我們在每次切換時，更靠近自己真正想活出的方向。

## 三分鐘上手：先讓系統活起來

第一次打開 NBL，頁面空空的是正常的。
因為你的狀態還沒有被宣告進系統。

可以先這樣開始。

1. 一開始，在 Guide 裡，把啟動預設頁面改成 Candidates。
2. 點 Scheduled，一開始 NBL會自動為你建立一條 `每日檢查 Inbox` 的條目。
3. 有想法就先丟 Inbox，先收集，不急著分類。

這個階段的目標很簡單。
先讓你在有 idea 的時候，習慣使用NBL快速記錄下來。

## 每天一次 Inbox 清理，把想法變狀態

每當你打開 NBL，若有到期或者需要注意的任務，會在 Candidates 顯示。
你可以直接開始該任務，系統就會開始計時，並提供終止或中斷能力。

建議每天固定一個時間做 Inbox 清理。您的預設的 `每日檢查 Inbox` 條目會在每天早上9:00以紅色條目的方式顯示在 Candidates 裡面，提醒你該去檢查 Inbox 了。

清理 Inbox 時，核心是把想法放到對的位置。

- 是資料參考：放 Resource
- 是有時效或週期：放 Scheduled
- 是會反覆做的主線行動：放 Task Pool
- 是一次性短任務：放 Micro Tasks

## 分類判準：先 Scheduled，再 Task Pool，最後 Micro Tasks

這個優先順序很重要，因為提醒機制不同。

Scheduled
- 適合時間點、週期性、鍊式活動
- 例如洗衣服後 60 分鐘提醒晾衣服

Task Pool
- 適合反覆執行、想看當天累積次數與時長的條目
- 結束後仍可在 Candidates 看到當日累積資訊

Micro Tasks
- 適合一次性短任務
- 通常完成後狀態轉為 Done

一句話判斷法。
先看有沒有時效，再看是否常態循環，最後才是一次性。

## 不只任務管理，也是習慣與情緒管理

NBL 的一個關鍵特性是：
你可以把狀態任務化，包括你不想花太多時間的狀態。

例如。

- ⛔ 發脾氣不超過十次與不超過 10 分鐘
- ⛔ 娛樂不超過 30 分鐘
- 🟢 試著在不能接受的狀態中找到可感恩的點，並且感恩，至少三次

當「壞習慣」也被如實記錄，改變才會開始發生。

你會逐漸看見。

- 我今天在哪些狀態停太久
- 我最常在哪些時間點偏離
- 哪些任務其實越做越順

這些都不是抽象感覺，而是可回顧的 Log。

## Local-first 與同步：不是監控，而是延續

NBL 採取的是先本地可用，再做同步的思路。

這代表即使沒有穩定網路，你仍可以先使用核心流程。
同步的目的也不是評分你，而是讓你的工作流可以跨裝置延續。

所以它比較像一個可攜式行為介面，而不是監控儀表板。

## 邊界與現況

NBL 能做的，是幫你把狀態切換看清楚。
NBL 不能做的，是替你做人生選擇。

目前不同平台的整合程度仍有差異，部分功能正在持續改善中。
但這不妨礙你現在就開始。

先記錄，先回顧，先做小幅調整。

## 目前到未來：AI 是建議層，你仍是主體

未來，AI 可以幫忙做分析與建議，提供更好的下一步選項。

但主權仍然在你。
AI 提醒，你選擇；AI 建議，你決定。

我相信最重要的，不是把自己訓練成機器。
而是在信實與愛裡，長期地、真實地，優化自己的生活流程。

## 結語：今天就做一個最小開始

今天只做兩件事。
點開 Scheduled，讓他自動產生每天早上9:00提醒你的 `每日檢查 Inbox` 條目。
然後，想想有甚麼想法都先寫到 Inbox 裡面去，先收集起來。

明天再做兩件事。
9:00後點開 Candidates，完整體驗一次一個任務的開始與結束的流程。這裡已經有一個任務 `每日檢查 Inbox`，就先從這個任務開始吧。
您也可以修改固定時間檢查 Inbox。
一樣，想想有甚麼想法都先寫到 Inbox 裡面去，先收集起來。

後天做第三件事。
在開始 `每日檢查 Inbox` 這個任務後，點 `📭` 圖案，跳到 Inbox 裡面去看看有哪些想法了。然後就開始分類吧。
這樣，你的 Candidates 就會越來越多了。

你不需要完美執行。
你只需要一次次回到主線。

## 投稿平台建議（作者備註）

如果以這篇內容的屬性來看，建議優先順序如下。

1. vocus
- 適合長文敘事、觀念鋪陳、系列化經營
- 讀者願意花時間讀完完整論述

2. Medium
- 適合技術與產品理念混合文
- 可接觸國際開發者讀者，建議另做英文版

3. 個人網站或部落格（例如 GitHub Pages）
- 最能累積可長期索引的主內容
- 可作為所有社群導流母站

4. Facebook 或 Threads（導流短文）
- 不建議當首發主平台
- 適合作為精簡版觀點，導流到 vocus 正文

實務上可採一主兩副。
主平台放 vocus 全文，副平台放 Medium 版本與社群短貼導流。
