# NonBlockingLife English Simple Script

> Purpose: a simple English script for a non-native speaker.  
> Rule: short sentences, easy words, clear pauses.

---

## 0) How to use this script

- Speak slowly.
- Do not rush.
- Pause after each sentence.
- Let the screen show the details.
- Your goal is clarity, not perfect accent.

---

## A) English Shorts Script (about 45 to 60 seconds)

### Version 1: Problem + Demo

You are not lazy.
You are just interrupted, and you lose your main track.

I am building a system called NonBlockingLife.

It is not just a todo app.
It is a task system for real life interruptions.

I use Event Loop thinking.
I have Inbox, Candidates, Task Pool, Scheduled, Micro Tasks, and Log.

If I suddenly think of something,
I put it in Inbox first.
I do not force a decision right away.

start one task,
get interrupted,
end it,
and log everything.

It is fast.

This is NonBlockingLife.

## B) English Main Video Script (about 3 to 4 minutes)

### [00:00 - 00:20] Problem

Hi, I am ychsue.

I used many todo tools before.
But my real problem was not the list.
My real problem was interruption.

After I got interrupted, I often could not return to the main task.
And later, I could not explain where my time went.

### [00:20 - 00:45] Core Idea

So I built NonBlockingLife.

The idea comes from Event Loop thinking.
I treat my attention like a single thread.
One thing at a time.

I separate tasks into:
Task Pool,
Scheduled,
Micro Tasks,
and Log.

### [00:45 - 01:10] System Structure

The current version is a PWA.
It is local-first.
That means daily actions are fast.

For sync and backup,
I use Google Apps Script and Google Sheets.
So the data stays under my control.

### [01:10 - 02:10] Demo

Let me show the main flow.

First, I open Candidates.
Then I refresh the candidate list.

This list is not random.
It is based on priority,
quota,
and scheduled timing.

Now I start one task.
The system writes a START log.

If I get interrupted,
I press interrupt.

In this system,
interruption is not failure.
It is a normal event.
It should be tracked.

Then I end the task.
The system records duration,
updates the state,
and writes to the log again.

### [02:10 - 02:40] Log

Here is the log page.

I can review recent records.
I can search by keyword.
I can see what I did,
when I was interrupted,
and how long each part took.

### [02:40 - 03:10] Sync

Now about sync.

Task Pool,
Scheduled,
Micro Tasks,
and Inbox
can sync with Google Sheets.

The log is mainly pushed to Google Sheets for analysis.

This is not live real-time sync.
It is a more controlled sync model.

### [03:10 - 03:35] iPhone workflow

I also use iPhone Shortcuts.

So I can trigger actions quickly,
then manage details in the PWA,
and review data later.

### [03:35 - 03:55] Ending

So this is NonBlockingLife.
It is not another todo app.
It is a personal scheduling system for real interruptions.

If you are interested,
please check the GitHub project.
Thank you.

## E) Recording tips for your English version

- Record line by line, not paragraph by paragraph.
- Keep each sentence short.
- Smile a little when speaking. It improves clarity.
- If you feel stuck, cut the sentence in half.
- English viewers usually forgive accent. They do not forgive unclear structure.

> Your English is good enough if the idea is clear. Clear beats fancy.
