# 中文

1. 痛點：
    1. 我以前最大的痛點，不是沒清單，而是這些清單散落在各個不同的APP甚至是裝置裡面，各有各的管理法。這也形成了一種負擔。
    2. 每當打岔後，想回來主線，腦袋常常在決策打開哪個APP時就卡住了，最後還是常常靠記憶力，或者跑去滑手機。
    3. 結果就是，一天很忙、很累，但晚上回想，時間到底花在哪裡，常常講不清楚。
2. 核心概念：
    1. NonBlockingLife 的做法是，以他為簡易的任務與想法的中控中心，複雜的管理法則則依然由各個專門的APP來做。
    2. 沒事時，看他的任務清單，立即就可選一個任務來進入工作狀態
    3. 進入工作狀態後，利用番茄時鐘法，把大腦當單執行緒，一次只專注做一件事。
    4. 無論何時，一有想法，直接丟inbox，以後處理，他的URL則連到對應的APP或網頁裡面。
    5. 隨時都可進入中斷狀態，這也是種任務，中斷結束後跟其他任務一樣紀錄。
    6. Task Pool 是主任務，Scheduled 是定時或回調任務，Micro Tasks 是碎任務，Log 則是完整事件流。
    7. 每日定時檢查 Inbox 裡面，可以直接使用 Move 來將Inbox條目變成這些任務。
    8. 這些任務，就會依條件排入任務清單供您選擇。
    9. 平常本機操作，速度快；同步到雲端時，資料在你自己的 Google Drive。
3. 可用平台：
   1. 由於這是個無須後端的網頁應用程式，理論上可以在任何支援現代瀏覽器的設備上運行。
   2. 因此，他的缺點就是，不像專門的APP那樣，許多手機原生應用才能操縱的手機功能，無法實現。
   3. 我的手機是iPhone，所以，我是透過 iPhone 的捷徑功能，來實現一些手機原生功能的整合，例如：快速新增任務到 Inbox。
   4. 這些Shortcuts 您可以由導讀頁面裡面下載，大部分就是呼叫API，所以，您若想自己寫也可以。
   5. 如果您是 Android 用戶，您一樣可透過API來快速到達某些功能，等我找到Android手機，我再來研究，或者您可以提供建議
   6. 如果是電腦用戶，則建議安裝成 PWA，這樣就像一般APP一樣，然後，我有提供 `web+nbl://` 的URL schema，您也可以自己寫一些桌面捷徑來呼叫特定功能。
4. 開源軟體：
   1. 這個專案是開源的，您可以在 GitHub 上找到源碼，歡迎提出問題或貢獻代碼。
   2. 由於這是一個個人專案，目前的開發和維護主要由我一人負責，但我非常歡迎社群的參與和支持。
