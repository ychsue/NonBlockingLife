# NonBlockingLife 拍攝當天 Checklist + 施行細節

> 目標：一人拍攝也能穩定產出。預設你要同時完成 60 秒 Shorts 與 5~6 分鐘主影片。

---

## 0) 當天總流程（建議時程）

- 09:00-09:30 場地與設備檢查
- 09:30-10:00 螢幕素材預演與重置資料
- 10:00-11:00 A-Roll（本人口播）
- 11:00-12:00 B-Roll（PWA 操作錄屏）
- 13:30-14:10 iPhone Shortcuts 錄屏
- 14:10-15:00 補鏡與收音備份
- 15:00-16:30 粗剪（先出主片）
- 16:30-17:10 切 Shorts
- 17:10-17:40 上字幕與封面
- 17:40-18:00 上片與發布文案

---

## 1) 前一天 Checklist（D-1）

- [ ] 口播稿最終版定稿（用 [11th_voiceover_script.md](11th_voiceover_script.md)）
- [ ] PWA 資料準備好示範用任務（Task Pool / Scheduled / Log）
- [ ] GAS URL 可正常 ping
- [ ] iPhone 內 Shortcuts 可觸發（QueryOptions / Interrupt / Inbox / Scheduled / NBL_Timer）
- [ ] 麥克風、耳機、充電器、延長線、行動電源就位
- [ ] 鏡頭擦拭、畫面背景清理
- [ ] 準備一份「錄壞可重錄」的心理預設

## 2) 拍攝當天 Checklist（開錄前）

### 場景與器材

- [ ] 光源穩定（臉部無明顯陰影）
- [ ] 收音測試 15 秒（有無噪音、爆音、底噪）
- [ ] 相機/手機固定，不手持晃動
- [ ] 畫面保留安全邊界（給字幕空間）

### 錄屏前置

- [ ] 瀏覽器放大比例固定（建議 110%-125%）
- [ ] 關閉非必要通知
- [ ] PWA 已切到乾淨狀態（避免隱私資訊）
- [ ] 準備好 demo 資料：
  - [ ] 一筆「臨時想法」可示範丟進 Inbox
  - [ ] 一個可開始的候選任務
  - [ ] 一個可展示的中斷場景
  - [ ] 至少 3 筆可看的 Log
  - [ ] 一筆 Scheduled（含 remind 或 callback）

### 內容節奏

- [ ] 每 8~12 秒有可見動作（切頁/點擊/狀態變化）
- [ ] 每段口播先停 1 秒再說
- [ ] 一段至少錄 2 take（穩定版 + 情緒版）

## 3) 鏡頭施行細節（Shot List）

## A-Roll（人像口播）

### Shot A1 開場痛點（15~20 秒）

- 內容：為什麼傳統待辦不夠
- Take：2
- 成功條件：語速穩、眼神直視鏡頭、語尾有力

### Shot A2 核心概念（20~30 秒）

- 內容：Event Loop 對應到人生調度
- Take：2
- 成功條件：Task Pool / Scheduled / Log 三詞清楚

### Shot A3 收尾 CTA（15 秒）

- 內容：開源、歡迎試用與回饋
- Take：2
- 成功條件：結尾語氣堅定，不拖尾

## B-Roll（螢幕操作）

### Shot B1 Guide 全貌（10 秒）

- 操作：打開 Guide，停 2 秒，再慢速捲動
- 成功條件：Shortcuts 區塊可清楚看到

### Shot B2 Candidates 主流程（35~45 秒）

- 操作：刷新候選 → 點任務 → 開始
- 成功條件：看得到任務狀態切換

### Shot B2-1 Inbox 快速捕捉（10~15 秒）

- 操作：新增一筆 Inbox（不用分類）
- 成功條件：觀眾理解 Inbox 是緩衝區，不是立即排程

### Shot B3 Interrupt 流程（20~30 秒）

- 操作：按中斷 → 再結束
- 成功條件：Log 有對應事件，且能口頭說清楚與 Inbox 的差異

### Shot B4 Sync 流程（15~20 秒）

- 操作：按同步、顯示成功訊息
- 成功條件：狀態文案可讀

### Shot B5 Log 回看（20 秒）

- 操作：切 7 天/30 天 + 搜尋關鍵字
- 成功條件：觀眾理解可追溯性

### Shot B6 Scheduled（15~20 秒）

- 操作：展示 cron/remind/callback 欄位
- 成功條件：說得出事件鏈概念

## iPhone 錄屏

### Shot I1 QueryOptions（10~15 秒）

- 成功條件：看得到從手機觸發流程

### Shot I2 Interrupt（10~15 秒）

- 成功條件：中斷情境清楚

### Shot I3 Inbox 或 Scheduled 快速輸入（10~15 秒）

- 成功條件：展現低摩擦輸入

## 4) 一人製作執行細節（防翻車）

### 錄口播技巧

- 每句最多兩個逗點，避免一口氣唸太長。
- 卡詞不用停機，回到上一句重唸即可。
- 重要詞加重音：本機優先、同步、中斷、回到主線。

### 錄屏技巧

- 滑鼠移動慢 20%，點擊後停 0.5 秒。
- 不要邊錄邊想，照 Shot List 走。
- 每段操作結束留 2 秒空白，方便剪接。

### 檔案管理

- 命名規則：
  - A-Roll：A_章節_第幾次.mp4
  - B-Roll：B_功能_第幾次.mp4
  - iPhone：I_功能_第幾次.mp4
- 備份：
  - [ ] 本機一份
  - [ ] 雲端一份

## 5) 後製 Checklist（同日完成版）

### 粗剪

- [ ] 先拼主片，再切 Shorts（不要反過來）
- [ ] 保留「動作點」前後各 0.3~0.5 秒
- [ ] 刪掉口頭禪與長停頓

### 字幕與視覺

- [ ] 字幕關鍵詞統一：START / INTERRUPT / END / SYNC / LOG
- [ ] 每段重點詞用一致色塊
- [ ] 不堆太多特效，保持可讀性

### 音訊

- [ ] 人聲音量一致
- [ ] 背景音樂壓低，不蓋口播
- [ ] 開場 3 秒抓耳、收尾 3 秒平穩

## 7) 口播防踩雷提示（拍攝時旁白提醒）

- 說明同步時，請講「手動同步為主」而非「完全即時同步」。
- 說明 AI 時，請講「加權候選與分析」而非「全自動決策」。
- 強調資料掌控：資料在使用者自己的 Google Drive。

## 8) 拍完 30 分鐘內就做的事

- [ ] 先丟出 1 版不完美粗剪（避免拖延）
- [ ] 馬上切 2 支 Shorts（痛點版 / 主流程版）
- [ ] 發佈後 24 小時內回覆前 10 則留言
- [ ] 收斂下一支影片題目（從留言挑）

> 原則：先完成，再優化。影片是產品迭代的一部分，不是一次定生死。
