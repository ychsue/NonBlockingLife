## 使用方式（開錄前 1 分鐘看）

- 先照稿唸一版「穩定版」，再錄一版「情緒版」。
- 每段前 1 秒停頓，方便後製剪接。
- 口播中的「畫面提示」不用唸出來，是給你對鏡頭用的。

## B) 5~6 分鐘主影片逐字稿（完整版）

## [00:00 - 00:25] 開場痛點

（畫面：你本人 + 手機通知 + 桌面待辦）  
嗨，我是 ychsue。  
我以前最大的痛點，不是沒清單，而是被打岔之後回不來。  
一天很忙，但晚上回想，時間到底花在哪裡，常常講不清楚。

## [00:25 - 00:55] 核心概念

（畫面：Event Loop 對應圖）  
NonBlockingLife 的做法是，把大腦當單執行緒。  
一次只做一件事，避免主線被阻塞。  
Task Pool 是主任務，Scheduled 是定時或回調任務，Micro Tasks 是碎任務，Log 則是完整事件流。

## [00:55 - 01:20] 系統總覽

（畫面：Guide 分頁）  
現在這套是 v2.0：PWA 本地優先，GAS 加 Google Sheets 當同步層。  
也就是說，日常操作在本機，速度快；同步到雲端時，資料在你自己的 Google Drive。

## [01:20 - 02:45] 主流程 Demo

（畫面：Task Pool → Selection Cache）  
我先切到 Candidates，按刷新候選。  
你會看到候選不是死排序，而是根據優先度、配額、排程狀態做加權。  

（畫面：Inbox 快速新增）  
如果我突然想到一件事，但現在不想切換上下文，  
我會先丟到 Inbox。  
Inbox 的角色是緩衝，不是立即決策。  

（畫面：點任務開始）  
我現在開始這個任務。  
系統會把當前任務掛到 Dashboard，並寫一筆 START 到 Log。  

（畫面：中斷流程）  
假設現在突然被電話打斷，我按中斷。  
中斷在這裡不是失敗，而是一段正式被追蹤的事件。  

（畫面：Inbox 與 Interrupt 對比圖卡）  
簡單說，Inbox 是「先記下來」，Interrupt 是「現在就得處理」。  

（畫面：結束流程）  
事情處理完我按結束，系統會計算 duration，更新任務狀態，然後再寫入 Log。

## [02:45 - 03:35] Log 回看

（畫面：Log 分頁，切天數篩選 + 搜尋）  
這裡是 Log。  
我可以看最近 1、3、7、14、30、90 天，還可以直接搜尋關鍵字。  
今天做了什麼、被打斷幾次、每段多久，都看得到。

## [03:35 - 04:20] 同步與還原

（畫面：右上 SyncStatus）  
接著是同步。  
Task Pool、Scheduled、Micro Tasks、Inbox 是雙向同步。  
Log 目前是單向推送到 Google Sheets，主要是給分析使用。  
如果你換裝置，也可以用還原功能，把本地清掉後重新拉回雲端資料。

## [04:20 - 05:00] iPhone Shortcuts 串接

（畫面：Guide 裡面的 Shortcuts 區塊 + iPhone 錄屏）  
我平常是 iPhone 觸發、PWA 管理、Sheets 分析。  
常用的有 QueryOptions、NBL Interrupt、NBL Inbox、NBL Scheduled，還有 NBL_Timer。  
所以在走路、通勤、會議前後，都可以快速進入流程。

## [05:00 - 05:35] 適合對象 + 誠實界線

（畫面：你本人）  
這套很適合常被打岔、想把任務狀態機化的人。  
如果你只想最簡單打卡，不想設定 Shortcuts 或同步，可能就太重了。  

## [05:35 - 05:55] 結尾 CTA

（畫面：GitHub + README）  
如果你也想把時間管理做成可觀測、可回復、可演進的系統，歡迎試試 NonBlockingLife。  
GitHub 開源中，歡迎 issue 與 PR。謝謝你。

## C) 12 分鐘教學版（章節逐字模板）

> 這版建議以「5~6 分鐘主片」為前半，後半補安裝與常見問題。

### [05:55 - 07:20] SetupWizard 流程

（畫面：SetupWizard）  
我快速示範第一次設定。  
先建立 Google Sheet，從 Sheet 開 Apps Script，貼上程式碼後部署成 Web App。  
把 URL 貼回 PWA，測試成功後就能同步。

### [07:20 - 08:30] Scheduled 深入

（畫面：Scheduled 表）  
這裡可以設定 Cron、remindBefore、remindAfter、callback。  
你可以把它當作生活裡的 callback queue。  
像洗衣結束後 60 分鐘提醒晾衣服，就是一個事件鏈。

### [08:30 - 09:30] 中斷哲學

（畫面：Selection Cache + Log）  
很多系統把中斷當失敗，但 NBL 把它當正常事件。  
因為真實生活一定有中斷，重點是可追蹤、可回線。

### [09:30 - 10:40] 同步策略講清楚

（畫面：SyncStatus + 說明圖卡）  
再強調一次：本地優先是速度，同步是跨裝置。  
目前不是即時雙向直播同步，而是以手動同步為主，穩定可控。

### [10:40 - 11:30] 常見 QA

Q：離線可用嗎？  
A：可，本機可持續操作。  
Q：資料在哪？  
A：本機 IndexedDB + 你自己的 Google Sheets。  
Q：會不會很難上手？  
A：先用 Task Pool + Candidates + Log 三步就能開始。

### [11:30 - 12:00] 願景收尾

我不是想做第 N 個待辦工具，  
而是做一個能承受現實摩擦、讓主線可恢復的個人調度系統。
