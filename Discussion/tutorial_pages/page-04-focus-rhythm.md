# Page 04: 專注節奏與番茄鐘

## 這一頁要傳達什麼

這一頁不是在講「怎麼選任務」，也不是在講「怎麼收集想法」，而是在講：一旦你選好了現在要做的事，Non-Blocking Life 會幫你把開始、專注、結束、休息接成一個節奏，而不是每次都靠意志力硬撐。

使用者看完這一頁，應該要感受到：

1. 任務一旦開始，就會進入一個明確的專注節奏。
2. 結束任務後，不需要自己再想休息多久，系統會自然切到休息段。
3. 這不是複雜的時間管理理論，而是很實際的開始與結束節奏。

## 與 Page 03 的分工

Page 03 已經回答了：

- 現在適合做什麼？
- 突發想法怎麼先收進 inbox？

Page 04 要回答的是：

- 一旦選好任務，怎麼開始？
- 開始之後，怎麼維持專注節奏？
- 結束之後，怎麼自然切到休息？

也就是說：

- Page 03 是中控與分流
- Page 04 是開始後的節奏管理

## 依照實際使用習慣收斂後的核心訊息

你描述的真實流程是：

1. 先用 action=query 看現在適合做什麼。
2. 選一個後，就開始任務。
3. 開始後進入專注。
4. 結束後切換到休息。

因此這一頁最適合講的，不是「iPhone 捷徑很好用」，而是：

開始與結束之間，系統幫你建立專注節奏。

iPhone 捷徑若要出現，應該只是輔助說明這個節奏可以被觸發得很快，而不是這一頁的主角。

## 畫面構圖建議

### 主視覺概念

畫面分成左右或上下兩段節奏：

1. 左側或上半部是「Focus 30」
畫面收斂、色彩單純、任務卡片被選中、計時器跑動，表示進入專注。

2. 右側或下半部是「Rest 10」
畫面稍微放鬆、顏色變柔和、計時器切換到休息模式，表示節奏自然轉場。

3. 中間有一個明確的切換點
例如 START -> FOCUS -> END -> REST 的四段流程。

### 畫面草圖

```text
  START        FOCUS 30          END        REST 10
   ▶             ⏱️               ■            ☕
   任務卡片      單色/收斂畫面      完成            放鬆色彩
```

## 動畫結構建議

### 段落 1：開始任務

- 一張被選中的任務卡片出現
- Start 按鈕或開始狀態亮起
- 畫面由多色切到較單純的專注色調

### 段落 2：30 分鐘專注

- 一個 30 分鐘的計時器或圓弧開始前進
- 畫面中的干擾元素退場或淡化
- 使用者感覺到「現在就是專注時間」

### 段落 3：結束任務

- End 狀態被觸發
- 專注區塊完成或打勾
- 計時器切換狀態

### 段落 4：10 分鐘休息

- 計時器轉到休息模式
- 畫面色調變柔和
- 顯示 10 分鐘休息段，讓節奏不是硬斷掉，而是自然過渡

## SVG 圖層建議

1. 任務卡片
2. Start / End 狀態標記
3. 主計時器或圓弧時鐘
4. Focus 30 標示
5. Rest 10 標示
6. 色彩切換背景層
7. 少量狀態 icon，例如 ▶、■、☕ 或 ✅

## 動畫時序

1. 0ms 到 600ms
任務卡片與開始狀態淡入。

2. 600ms 到 2600ms
30 分鐘專注段啟動，計時器穩定前進。

3. 2600ms 到 3200ms
End 被觸發，畫面從專注狀態切換。

4. 3200ms 到 5000ms
10 分鐘休息段啟動，畫面色調轉柔和。

5. 5000ms 到 5600ms
整體停住，讓使用者理解這不是一次性衝刺，而是一種可持續節奏。

## 視覺語氣

- 專注段應該簡潔、穩定、內收
- 休息段應該放鬆、透氣、柔和
- 不要做得像壓力很大的 productivity app
- 重點是「自然切換節奏」，不是「拼命倒數」

## 文案方向

這頁的文字建議聚焦這句：

選好任務之後，剩下的就是開始、專注、結束、休息。

你不用每次都重新決定下一步的節奏。

## iPhone 捷徑在這一頁的角色

iPhone 捷徑可以出現，但只建議作為次要輔助，例如：

- 一個小小的手機或捷徑 icon 在角落，表示這個節奏可以被快速觸發
- 不要讓它搶走主畫面焦點

真正要講 iPhone 捷徑細節，還是放到延伸版頁面比較適合。

## 與其他頁面的關係

- Page 01: 主線被打斷
- Page 02: NBL 幫你回到主線
- Page 03: NBL 是中控中心，幫你決定與收集
- Page 04: 選定任務後，進入專注與休息節奏
- Page 05: 同步與安心感

## 結論

Page 04 應該是一頁「節奏感」很強的頁面。它的任務不是教功能，而是讓使用者理解：Non-Blocking Life 不只是把任務列出來，而是幫你把開始與休息接成一個可持續的工作節奏。
