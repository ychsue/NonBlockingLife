# Page 05: Google Sheets 備份、同步與分析

## 這一頁要傳達什麼

這一頁不是要教使用者立刻完成 Google Sheets 與 GAS 的技術設定，而是要先讓使用者理解：

1. 你的資料不會被鎖在 App 裡。
2. 你可以把資料同步到你自己的 Google Sheets。
3. 這樣做的價值是備份、跨裝置延續，以及之後可自行分析。

因此這一頁的核心不是「怎麼設」，而是「為什麼值得設」。

## 與 Page 06 的分工

Page 05 回答的是：

- 為什麼要接 Google Sheets？
- 接上後帶來什麼安心感與延展性？

Page 06 才負責回答：

- 第一次開始使用時，要先建立哪些資料？
- 接下來應該做的第一步是什麼？

也就是說：

- Page 05 是同步價值與安心感
- Page 06 是實際開始使用前的行動引導

## 核心訊息收斂

這頁建議聚焦這句：

你的資料可以同步到你自己的 Google Sheets，所以更安心，也更容易跨裝置延續與後續分析。

這裡最重要的是「你自己的」三個字。

因為這能同時傳達三件事：

1. 可備份
2. 可攜帶
3. 可分析

## 關於 Google Sheets 與 GAS icon

現階段不建議把 Page 05 綁死在官方 icon 上。

原因是：

1. 這頁的主角其實不是品牌 logo，而是資料流向與安心感。
2. 先用抽象化的 SVG 元素就足夠傳達概念，例如：
   - 一個本地資料卡片堆
   - 一條同步線
   - 一個代表試算表的綠色表格框
   - 一個代表雲端腳本橋接的小節點
3. 若之後你真的要讓品牌辨識更強，再由你提供 icon 素材即可。

因此我的建議是：

- 規劃與 MVP 動畫階段：不必先下載 icon
- 若你之後想做更明確的品牌辨識：再由你提供合法素材或自行放入工作區

## 關於使用者必須自己設定這件事

這一點非常重要，而且我認為應該直接承認，而不是藏起來。

這頁可以明確傳達：

- Google Sheets 與 GAS 需要由使用者自己建立與填入
- 但這不是缺點，反而能強化「資料在你自己手上」

換句話說，這頁不應該把它講成「麻煩的技術設定」，而要講成：

這是一個把資料掌握權交還給使用者的設計。

## 影片教學的角色

你打算另外拍影片示範設定流程，我認為這是對的。

而且這樣剛好能讓 onboarding 分工更乾淨：

1. Page 05 負責建立價值感與信任感
2. 影片負責手把手操作

這比把一堆設定細節塞進首次輪播好很多。

這頁甚至可以在文案或畫面角落暗示：

- 設定步驟可看教學影片
- 首次輪播只先告訴你，資料是可以同步到你自己的表

## 畫面構圖建議

### 主視覺概念

畫面可以分成三個區塊，但同時運動元素仍控制在 4 個以內：

1. 左側是本地資料
用幾張卡片或幾個小資料列，代表 inbox、tasks、scheduled、log 等資料仍在本機先被使用。

2. 中間是同步橋
一條穩定的連接線，或一個小型橋接節點，代表同步流程。

3. 右側是使用者自己的 Google Sheets
用一個綠色表格框表示試算表，旁邊可再用少量圖示表示 backup / sync / analysis。

### 畫面草圖

```text
  Local Data       Sync Bridge        Your Google Sheet
  □ inbox          ----●----          ┌──────────────┐
  □ tasks                            │ sheet data   │
  □ scheduled                        │ backup sync  │
                                      └──────────────┘
```

## 動畫結構建議

### 段落 1：本地資料先出現

- 左側數張資料卡片淡入
- 表示日常操作先在本地完成

### 段落 2：同步橋接亮起

- 中間連接線被點亮
- 一兩個小資料 token 穩定地往右移動
- 節奏不要急，不要像大量資料狂噴

### 段落 3：Google Sheets 收到資料

- 右側綠色表格框亮起
- 表格裡新增一兩行
- 畫面旁出現 backup / sync / analyze 三個小標記中的兩到三個

### 段落 4：停住讓使用者理解

- 整體停住
- 讓使用者讀懂這不是「雲端帳號綁死」，而是「同步到自己的資料表」

## SVG 圖層建議

1. 左側本地資料卡片 3 張
2. 中間同步線與橋接節點 1 組
3. 右側綠色表格框 1 個
4. 表格內新增的資料列 2 到 3 條
5. 小標記 backup / sync / analysis

## 動畫時序

1. 0ms 到 600ms
本地資料卡片與 Google Sheet 框體淡入。

2. 600ms 到 2200ms
同步橋接亮起，一兩個 token 往右移動。

3. 2200ms 到 3800ms
Google Sheet 內新增資料列，backup / sync / analysis 小標記依序亮起。

4. 3800ms 到 5200ms
畫面停住，讓使用者吸收「資料在自己手上」這件事。

## 視覺語氣

- 應該給人穩定、可信、安心的感覺
- 不要做成複雜的工程架構圖
- 不要一開始就出現太多 API、表名或部署細節
- 整體比 Page 03、Page 04 更安定，少一點動態焦慮

## 文案方向

這頁文字建議聚焦這句：

你可以把資料同步到你自己的 Google Sheets，方便備份、跨裝置延續與後續分析。

第二句可以補：

設定需要你自己建立表與腳本，但也因此，資料掌握權在你手上。

## 與影片教學的搭配方式

這頁不需要把教學影片內容完整塞進來。

比較好的策略是：

1. 輪播先建立「為什麼值得設定」
2. 說明頁或外部教學影片再補「怎麼設定」

這樣使用者比較不會在第一次接觸時就被技術步驟嚇走。

## 與其他頁面的關係

- Page 03: 中控中心，幫你收集與分流
- Page 04: 幫你建立開始後的專注節奏
- Page 05: 幫你把資料同步到自己的 Google Sheets，帶來安心感與延展性
- Page 06: 第一次實際開始使用時的行動引導

## 結論

Page 05 應該是一頁「安心感」頁面，而不是「設定教學」頁面。

你不需要現在就先去找 Google Sheets 或 GAS 的 icon；先把這頁做成抽象但清楚的同步視覺會更穩。至於實際設定流程，交給你另外拍的影片來處理，這樣首次輪播與教學影片的角色就分得很清楚。
