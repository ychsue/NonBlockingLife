# Page 06: 初次開始使用

## 這一頁要傳達什麼

這一頁不是再介紹產品價值，而是要把首次輪播收在一個非常明確的行動上：

1. 先去建立 Task Pool
2. 再去建立 Scheduled
3. 只要兩邊都不是空的，這個首次教學就不會再自動跳出

這一頁的任務不是說服，而是收尾與起步。

## 與前一頁的分工

Page 05 已經回答：

- 你可以把資料同步到自己的 Google Sheets
- 這帶來備份、延續與分析的安心感

Page 06 要回答的是：

- 現在就開始的第一步是什麼？
- 要做哪兩件最小必要動作？

也就是說：

- Page 05 是安心感
- Page 06 是起跑點

## 核心訊息收斂

這頁建議聚焦這句：

開始前，先在 Task Pool 與 Scheduled 各新增一筆資料。

第二句可以直接補：

完成這兩步後，首頁教學就不會再自動跳出。

## 畫面構圖建議

### 主視覺概念

畫面應該比前幾頁更直接，像一個啟動面板：

1. 左邊一張 Task Pool 卡
2. 右邊一張 Scheduled 卡
3. 中間或下方一個 Ready / Start 區塊

兩張卡片各自完成後，連到中間的開始狀態。

### 畫面草圖

```text
  [ Task Pool ]      [ Scheduled ]
       ✓                  ✓
         \              /
          \            /
           [ Ready to Start ]
```

## 動畫結構建議

### 段落 1：兩張必要卡片出現

- Task Pool 與 Scheduled 卡片淡入
- 卡片上有新增提示

### 段落 2：完成狀態亮起

- 兩張卡片各自出現完成勾勾或填入狀態
- 表示這兩邊都至少要有一筆資料

### 段落 3：連到開始狀態

- 兩條線匯流到下方或中央的 Ready 狀態
- Ready 區塊亮起

### 段落 4：教學停止提示

- 補一句輕量提示：完成後首次教學不再自動跳出
- 不要做得像警告，而是像完成啟動流程

## SVG 圖層建議

1. Task Pool 卡片
2. Scheduled 卡片
3. 兩個完成勾勾
4. 匯流連線
5. Ready / Start 區塊
6. 教學停止提示文字

## 動畫時序

1. 0ms 到 700ms
兩張卡片淡入。

2. 700ms 到 1800ms
卡片內的新增狀態與完成勾勾出現。

3. 1800ms 到 3000ms
兩條連線往中央或下方匯流。

4. 3000ms 到 4200ms
Ready 狀態亮起。

5. 4200ms 到 5200ms
完成提示停住，讓使用者知道下一步就是去新增資料。

## 視覺語氣

- 明確
- 輕巧
- 像啟動流程，而不是考試清單
- 要有收尾感，但不要有壓迫感

這頁的文字建議聚焦這句：

開始前，先在 Task Pool 與 Scheduled 各新增一筆資料。

補充句可以是：

## 與畫面按鈕的關係

這一頁很適合直接搭配兩個 CTA：

1. 前往 Task Pool
2. 前往 Scheduled

也就是說，這頁不只是說明頁，也是首次輪播的出口頁。

## 結論

Page 06 應該是一頁很乾淨的收尾頁。它不再增加新概念，而是把整個 onboarding 收斂成一個最小可行起步：先建兩筆資料，然後正式開始使用。
