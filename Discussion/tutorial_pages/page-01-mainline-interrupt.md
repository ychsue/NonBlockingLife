# Page 01: 主線被打斷

## 這一頁要傳達什麼

這一頁的核心不是「很忙」，而是「原本有主線，但在推進一半時不斷被打斷，最後力氣耗散到混亂的支線」。

使用者看完這一頁，應該立刻理解：

1. 問題不是自己不努力。
2. 問題是注意力與時間被切碎。
3. 這個 App 想處理的是「回到主線」這件事。

## 一眼看到的畫面

方形畫面中央偏上一條由左往右的空心粗箭頭，終點是一面偏灰色的旗子，代表原本要前進的人生主線。

箭頭前半段上方有一個時鐘，時鐘裡的黃色與紅色扇形逐漸擴張；同時，箭頭內部的黃色進度條慢慢填到一半。

當進度停住時，一個電話符號突然大幅出現，再縮小壓在中途節點上。接著，一條較粗的直線往下分出去，底端出現一個 🙋。然後第三條線直接從第二條線的終點往右延伸，沿路擠著冒出工作與娛樂混合的打岔圖示，最後落到右側更靠邊的小型亂線終點。

## 畫面構圖

### 主視覺區域

- 主箭頭：橫向置中但整體往上移，佔畫面寬度約 70% 到 78%
- 旗子：位於右上終點，保持靜態
- 時鐘：位於主箭頭前半段上方，略偏左中
- 中斷點：位於主箭頭大約 45% 到 50% 的位置，電話最後停在這裡
- 向下分支：從中斷點往下延伸到底部中間區域
- 雜事支線：從向下分支的終點往右延伸到右下區域
- 混亂終點：位於更靠右的位置，尺寸接近旗子，不要做得過大

## SVG 圖層拆解

### Layer 1: 背景底板

- 淡暖色背景
- 可保留很淡的放射狀光感
- 不需要動畫

### Layer 2: 黯淡主線粗箭頭

- 使用 SVG polygon
- 只保留外框，不做本體填色
- 先淡入
- 代表原本明確的主線方向

### Layer 3: 終點旗子

- 靜態小旗子，顏色偏灰
- 在最終目標位置
- 不需要動畫，只需先淡入

### Layer 4: 時鐘外框

- 簡單圓形即可
- 保持簡潔，避免真實鐘面過細

### Layer 5: 黃色扇形

- 從小角度擴張到約 180 度
- 代表投入的時間與精力

### Layer 6: 紅色扇形

- 黃色過半後才開始出現
- 由黃色的 180 度終點開始往後擴張
- 代表疲勞、壓力與拖延感加重

### Layer 7: 主箭頭內部進度條

- 黃色 bar 從左往右填滿
- 高度與箭頭同高，先填滿整個前段
- 停在箭頭中段，不可到終點
- 停止本身要讓人有「卡住」感

### Layer 8: 電話事件

- 使用 📞
- 先放大出現，再縮小停在中斷點旁
- 這是整頁最明確的轉折提示

### Layer 9: 向下分支線

- 比前一版更粗，且不使用圓角
- 顏色可延續黃色，但稍淡
- 由上往下長出

### Layer 10: 小人圖示

- 使用 🙋，表示有人來聊天或主動把注意力拉走
- 在分支底端出現
- 不需要位移很多，只要淡入或小幅上浮

### Layer 11: 往右雜事支線

- 從向下分支的終點再往右延伸
- 線條比前一版更粗，且不使用圓角
- 視覺上要和主線形成對比

### Layer 12: 雜事與娛樂節點

- 採混合型，第一版可先用 📺 💬 🍿 📦 這類工作與娛樂混在一起的節點
- 不要一次全部噴出
- 應該由疏到密、由慢到快

### Layer 13: 混亂終點

- 採亂線 SVG，而不是單純 emoji
- 大小與旗子接近，位置再靠右一些
- 這裡不追求精緻，先讓結論成立

## 動畫時序

### 版本：MVP

1. 0ms 到 500ms
主箭頭與旗子淡入。

2. 300ms 到 2200ms
黃色扇形慢慢擴張；同時箭頭內部進度條同步往前填。

3. 1600ms 到 2800ms
紅色扇形開始擴張，黃色進度條接近中段。

4. 2300ms
進度條停住。

5. 2400ms 到 3000ms
電話大幅出現，再縮小停到中斷點旁邊。

6. 3000ms 到 3800ms
向下分支線長出，小人淡入。

7. 3600ms 到 5200ms
往右的雜事支線延伸，雜事節點逐步出現。

8. 5200ms 到 6200ms
混亂終點出現，整體停住讓使用者看懂畫面。

## 動畫語氣

- 前半段應該是穩定、可預期、略帶希望
- 電話出現是第一個明顯打斷點
- 後半段節奏變快、變碎、變亂
- 最後不是爆炸式結尾，而是「怎麼會變成這樣」的耗散感

## 建議技術做法

### 建議使用

- inline SVG
- CSS keyframes
- transform
- opacity
- stroke-dasharray / stroke-dashoffset
- 少量 emoji 作事件節點

### 先不要使用

- Lottie
- JS timeline library
- 沿 path 精準移動的複雜物件動畫
- 粒子特效

## MVP 與 Enhanced 分界

### MVP 先做

- 主箭頭
- 旗子
- 時鐘外框
- 黃色扇形
- 紅色扇形
- 箭頭進度條
- 電話跳出
- 向下分支
- 小人
- 3 到 5 個雜事節點
- 混亂終點

### Enhanced 再考慮

- 更細的鐘面刻度
- 雜事節點帶有不同出現節奏
- 支線更自然的彎曲 path
- 更細緻的混亂終點圖形
- 首尾更流暢的 easing 微調

## 風險與注意事項

1. 如果同時做太多動畫，手機效能會先出問題。
2. emoji 跨平台外觀差異大，所以核心幾何一定要用 SVG。
3. 若畫面太複雜，使用者可能看不懂主訊息，所以要把主線、中斷點、混亂終點的對比做得很清楚。

## 實作前要再確認的點

1. 電話圖示已定為 📞。
2. 小人圖示已定為 🙋。
3. 雜事節點先採混合型。
4. 混亂終點已定為亂線 SVG。

## 結論

這一頁的關鍵不是技術難度，而是敘事清楚。只要主線、中斷、分流、混亂這四段被看懂，第一頁就成功了。後續實作時，應優先保證這四段訊息成立，再追求更漂亮的動畫細節。
